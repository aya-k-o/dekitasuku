# 要件定義書

## 1. プロジェクト概要

### アプリ名
できたすく（Dekitasuku）

### 目的
子どもの毎日の小さな頑張りを記録し、家族の宝物として残すためのタスク管理＆日記アプリ

### ターゲットユーザー
- **子ども**：小学生（タスクを達成してポイントを貯める、日記を書く）
- **親**：子どもの様子を確認し、返信を通じてコミュニケーションを取る

---

## 2. 機能要件

### 2.1 認証機能

#### ユーザー登録
- ユーザー名（ニックネーム）とパスワードで登録
- メールアドレス・本名・電話番号は収集しない（プライバシー配慮）
- パスワードは8文字以上
- 同一ユーザー名の重複登録を防止

#### ログイン
- ユーザー名とパスワードで認証
- ログイン成功時にセッションIDを再生成（セッション固定化攻撃対策）
- エラーメッセージはユーザー名・パスワードどちらが間違いか教えない

#### ログアウト
- セッション変数・セッションクッキーを完全に破棄

---

### 2.2 子ども側機能

#### タスク管理
- 今日のタスク一覧表示（ログイン中のユーザーの子どもだけ表示）
- 「できた！」ボタンでタスク達成を記録
- 達成済みタスクは「やったね！」表示（グレーアウト）
- 1日1回制限（同じタスクを同日に重複達成できない）

#### ポイントシステム
- タスク達成時にポイント獲得
- 累計ポイントを画面上部に常時表示
- ポイント履歴の確認

#### 日記機能
- 体調（😢😕😊😄🤩）と心の調子を5段階で選択
- 自由記述欄（任意）
- 1日1回記録
- 月ごとの日記一覧表示（「きろく」画面）
- 親からの返信を確認

#### ナビゲーション
- 「きょう」タブ：今日のタスクと日記
- 「きろく」タブ：過去の日記一覧
- 「ログアウト」：セッション破棄してログイン画面へ

---

### 2.3 親側機能（管理画面）

#### 認証
- ユーザー名・パスワード認証でログイン
- セッション管理
- ログアウト機能

#### 今日の様子確認
- ログイン中ユーザーの子どもだけ表示
- 全員分の今日のタスク達成状況を一覧表示
- 全員分の今日の日記を一覧表示
- その場で返信を書き込み可能

#### タスク管理
- タスクの登録・削除
- 子どもごとにタスクを設定
- タスクごとにポイント設定

#### 子ども管理
- 子どもの追加・削除
- 名前の確認・累計ポイントの確認

#### 日記管理
- 全期間の日記一覧表示
- 返信の確認・送信

#### ログ確認
- タスク達成履歴

---

## 3. 非機能要件

### 3.1 セキュリティ
- XSS対策：全ての出力に`htmlspecialchars()`適用（h関数として共通化）
- SQLインジェクション対策：PDOプリペアドステートメント使用
- パスワード管理：`.env`ファイルで管理、GitHubには非公開
- パスワードハッシュ化：`password_hash()` / `password_verify()`使用
- セッション管理：全ページでセッション確認、未認証はリダイレクト
- セッション固定化攻撃対策：ログイン成功時に`session_regenerate_id(true)`
- 認可チェック：`user_id`でデータの所有者確認、他の家族のデータへのアクセス不可

### 3.2 データ保護
- 論理削除：全テーブルに`deleted_at`カラムを実装
- データ復元可能性の確保
- 履歴の永続的保存
- 個人情報の最小化（メールアドレス・本名・電話番号を収集しない）

### 3.3 ユーザビリティ
- 子ども向けUI：カラフルな配色、顔アイコンによる直感的な操作
- モバイルフレンドリー：タブレット・スマホでの利用を想定
- シンプルなナビゲーション：2タブ構成

### 3.4 パフォーマンス
- トランザクション処理：ポイント加算とタスク記録を1トランザクションで実行
- インデックス設定：検索頻度の高いカラムに設定

---

## 4. 技術要件

### 4.1 開発環境
- Docker / Docker Compose
- PHP 8.2
- MySQL（MariaDB）
- phpMyAdmin

### 4.2 本番環境
- Xserver
- MySQL 8.0

### 4.3 使用技術
- バックエンド：PHP（素のPHP、フレームワーク不使用）
- フロントエンド：HTML / CSS / JavaScript
- データベース：MySQL

---

## 5. 制約事項

### 5.1 技術的制約
- フレームワーク不使用（学習目的のため、素のPHPで実装）
- 子どもはユーザー選択式（子ども専用のログインは設けない）

### 5.2 機能的制約
- ご褒美交換機能は未実装（将来的に実装予定）
- 効果音機能は未実装（将来的に実装予定）
- ログイン試行回数制限は未実装（将来的に実装予定）
- CSRFトークンは未実装（将来的に実装予定）

---

## 6. 今後の展望

- 効果音の追加（タスク達成時の「やったね！」効果音）
- ご褒美交換機能の実装
- 管理画面での月選択フィルター

## 3. 非機能要件

### 3.1 セキュリティ
- XSS対策：全ての出力に`htmlspecialchars()`適用（h関数として共通化）
- SQLインジェクション対策：PDOプリペアドステートメント使用
- パスワード管理：`.env`ファイルで管理、GitHubには非公開
- パスワードハッシュ化：`password_hash()` / `password_verify()`使用
- セッション管理：全ページでセッション確認、未認証はリダイレクト
- セッション固定化攻撃対策：ログイン成功時に`session_regenerate_id(true)`
- 認可チェック：`user_id`でデータの所有者確認、他の家族のデータへのアクセス不可

### 3.2 データ保護
- 論理削除：全テーブルに`deleted_at`カラムを実装
- データ復元可能性の確保
- 履歴の永続的保存
- 個人情報の最小化（メールアドレス・本名・電話番号を収集しない）

### 3.3 ユーザビリティ
- 子ども向けUI：カラフルな配色、顔アイコンによる直感的な操作
- モバイルフレンドリー：タブレット・スマホでの利用を想定
- シンプルなナビゲーション：2タブ構成

### 3.4 パフォーマンス
- トランザクション処理：ポイント加算とタスク記録を1トランザクションで実行
- インデックス設定：検索頻度の高いカラムに設定
